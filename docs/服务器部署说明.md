# 服务器部署说明

## 当前线上信息

- 域名：`quant.lmpai.online`
- 入口层：`Caddy`
- 前端内部端口：`127.0.0.1:3101`
- 后端内部端口：`127.0.0.1:8100`
- 部署目录：`/home/ubuntu/quant-research-workbench`
- 数据库：`/home/ubuntu/quant-research-workbench/backend/data/workbench.db`

## 当前服务

- 后端服务：`quant-research-api.service`
- 前端服务：`quant-research-web.service`
- 入口服务：`caddy.service`

## 代码更新

```bash
cd /home/ubuntu/quant-research-workbench
git fetch origin
git reset --hard origin/main
```

## 后端依赖更新

```bash
cd /home/ubuntu/quant-research-workbench
python3 -m venv backend/.venv
backend/.venv/bin/pip install --upgrade pip
backend/.venv/bin/pip install -r backend/requirements.txt
```

## 前端构建

```bash
cd /home/ubuntu/quant-research-workbench
npm --prefix frontend install
npm --prefix frontend run build
```

## 重启服务

```bash
sudo systemctl restart quant-research-api.service
sudo systemctl restart quant-research-web.service
sudo systemctl reload caddy
```

## 查看状态

```bash
sudo systemctl status quant-research-api.service --no-pager
sudo systemctl status quant-research-web.service --no-pager
sudo systemctl status caddy --no-pager
```

## 查看日志

```bash
sudo journalctl -u quant-research-api.service -n 100 --no-pager
sudo journalctl -u quant-research-web.service -n 100 --no-pager
sudo journalctl -u caddy -n 100 --no-pager
```

## 常见故障排查

### 1. 页面能打开，但接口报错

先检查：

```bash
curl http://127.0.0.1:8100/api/health
sudo systemctl status quant-research-api.service --no-pager
```

### 2. 域名打不开，但本机端口正常

先检查：

```bash
sudo systemctl status caddy --no-pager
sudo journalctl -u caddy -n 100 --no-pager
```

### 3. 前端改了代码但线上没变化

通常是没重新构建：

```bash
cd /home/ubuntu/quant-research-workbench
npm --prefix frontend run build
sudo systemctl restart quant-research-web.service
```

### 4. 后端改了代码但接口没变化

通常是没重启服务：

```bash
sudo systemctl restart quant-research-api.service
```
